# Frontend Application Requirements

---

## Next.js, TypeScript, AI Integration Frontend Application Requirements

**Project Type:** E-Commerce Platform

---

## 1. Technology Requirements

Your project must include the following technologies:

### Stack

- **Next.js (Recommended) or React**
- **TypeScript**
- **Tailwind CSS / CSS / Sass**
- **Responsive Design**

### AI Feature (Minimum 1)

Examples:

- AI content generator
- AI chatbot
- AI recommendation system
- AI search assistant
- AI review summary
- AI smart suggestions

Example:

- AI suggests travel destinations
- AI summarizes product reviews
- AI chatbot for customer help

---

## 2. Global UI & Design Rules

Follow a **clean and consistent UI design** across the entire project.

Rules:

- Use **maximum 3 primary colors**
- Optional **neutral colors (gray / white / black)**
- Support **Light Mode and Dark Mode**
- Maintain **consistent spacing, layout, and alignment**
- All cards must have:
  - Same height
  - Same width
  - Same border radius
  - Same visual style

Forms must include:

- Validation
- Error messages
- Success messages
- Loading indicators

The project must be **fully responsive**:

- Mobile
- Tablet
- Desktop

**No dummy text or placeholder content allowed**

---

## 3. Home / Landing Page

### Navbar

Requirements:

- Full-width navbar
- Minimum **4 routes when logged out**
- Minimum **6 routes when logged in**
- At least **one advanced menu**
  - Dropdown menu
  - Profile menu
- Sticky or fixed navbar
- Fully responsive

Example routes:

- Home
- Explore
- About
- Contact
- Dashboard
- Blog

### Hero Section

Requirements:

- Height limited to **60–70% of screen**
- Must include:
  - Call-to-Action button
  - Animation / slider / interactive UI
- Clear visual flow to the next section

### Landing Page Sections

Minimum **8 meaningful sections**.

Examples:

- Features
- Services
- Categories
- Popular Items
- Statistics
- Testimonials
- Blog
- FAQ
- Newsletter
- Call to Action

You can design **your own sections based on your project idea**.

### Footer

Footer must include:

- Working navigation links
- Contact information
- Social media links
- Copyright text

Only **working links allowed**

# Backend Server Requirements

---

## Express, TypeScript, Mongoose and AI Integration Backend Server Requirements

---

## 1. Technology Stack

Backend must use:

- **Runtime:** Node.js
- **Framework:** Express.js
- **Language:** TypeScript
- **Database:** MongoDB
- **ODM:** Mongoose
- **Authentication:** JSON Web Token
- **Password Hashing:** bcrypt
- **AI Integration:** Google Gemini API or similar

---

## 2. Core Backend Modules

### A. Authentication Module

Features:

- User Registration
- Login
- JWT authentication
- Password hashing
- Role-based authorization

Roles:

- USER
- ADMIN

Endpoints:

```
POST /api/auth/register
POST /api/auth/login
POST /api/auth/refresh-token
```

---

### B. User Module

Features:

- View profile
- Update profile
- Admin manage users

Endpoints:

```
GET    /api/users
GET    /api/users/:id
PATCH  /api/users/:id
DELETE /api/users/:id
PATCH  /api/users/role
```

---

### C. Item Module (Universal Listing)

Represents platform entities such as:

| Platform    | Item        |
|-------------|-------------|
| E-commerce  | Product     |
| Event       | Event       |
| Travel      | Destination |
| Food        | Restaurant  |
| Real Estate | Property    |

Schema example:

```
title
description
image
price
rating
location
category
createdBy
createdAt
```

Endpoints:

```
POST   /api/items
GET    /api/items
GET    /api/items/:id
PATCH  /api/items/:id
DELETE /api/items/:id
```

---

### D. Search, Filter & Pagination

**Search**

```
GET /api/items?search=keyword
```

Search fields:

- title
- description
- category

**Filtering**

```
GET /api/items?category=travel
GET /api/items?priceMin=10&priceMax=100
GET /api/items?rating=4
```

**Sorting**

```
GET /api/items?sort=price
GET /api/items?sort=-rating
GET /api/items?sort=-createdAt
```

**Pagination**

```
GET /api/items?page=1&limit=10
```

Response:

```json
{
  "data": [],
  "meta": {
    "page": 1,
    "limit": 10,
    "total": 120
  }
}
```

---

### E. Review Module

Users can rate and review items.

Schema:

```
rating
comment
userId
itemId
```

Endpoints:

```
POST   /api/reviews
GET    /api/reviews/item/:itemId
DELETE /api/reviews/:id
```

---

### F. Booking / Order Module

Schema:

```
userId
itemId
quantity
price
status
```

Endpoints:

```
POST   /api/bookings
GET    /api/bookings
PATCH  /api/bookings/:id
DELETE /api/bookings/:id
```

---

### G. Dashboard Analytics

Admin dashboard must return real statistics.

Endpoint:

```
GET /api/dashboard/stats
```

Response:

```json
{
  "totalUsers": 1200,
  "totalItems": 430,
  "totalOrders": 210,
  "totalRevenue": 35000
}
```

Chart data:

```
GET /api/dashboard/chart-data
```

Used for:

- Bar Chart
- Line Chart
- Pie Chart

---

### H. AI Module (Required)

Integrate AI using Google Gemini API.

Example features:

#### 1. AI Chatbot

```
POST /api/ai/chat
```

Users can ask questions like:

- Suggest travel destinations
- Recommend restaurants

#### 2. AI Content Generator

```
POST /api/ai/generate-description
```

Example:

- Input:

```json
{ "title": "Beach Resort" }
```

- Output: AI generated description

#### 3. AI Review Summary

```
POST /api/ai/review-summary
```

- Input: list of reviews
- Output: summarized feedback

---

```json
{
  "success": true,
  "message": "Request successful",
  "data": []
}
```

**Error:**

```json
{
  "success": false,
  "message": "Something went wrong"
}
```
